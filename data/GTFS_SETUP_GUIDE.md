# GTFS Setup Guide for EcoRoute-UVA

## Quick Start

### 1. Move GTFS ZIP Files to `data/gtfs-raw/`

These three files currently at the project root need to be moved:

```bash
# From the project root:
mv Charlottesvilleareatransit_GTFS.zip data/gtfs-raw/
mv University_Transit_Service_GTFS.zip data/gtfs-raw/
mv jaunt_connect_charlottesville.zip data/gtfs-raw/
```

**After moving, your `data/` directory should look like:**
```
data/
├── gtfs-raw/
│   ├── Charlottesvilleareatransit_GTFS.zip
│   ├── University_Transit_Service_GTFS.zip
│   └── jaunt_connect_charlottesville.zip
└── gtfs-parsed/
    ├── cat-gtfs.json          (generated by parse script)
    ├── uva-gtfs.json          (generated by parse script)
    └── jaunt-gtfs.json        (generated by parse script)
```

### 2. Parse GTFS Feeds (One-time setup)

After moving files, parse them into JSON:

```bash
npm run parse-gtfs
```

This:
- Reads ZIP files from `data/gtfs-raw/`
- Parses `stops.txt`, `routes.txt`, `stop_times.txt`, `calendar.txt`
- Generates `uva-gtfs.json`, `cat-gtfs.json`, `jaunt-gtfs.json` in `data/gtfs-parsed/`
- Each file includes stop locations, route info, and departure times by day of week

### 3. Commit Parsed JSON (not raw ZIPs)

```bash
git add data/gtfs-parsed/*.json
git commit -m "chore: add parsed GTFS data for transit routing"
```

The raw ZIP files (in `data/gtfs-raw/`) are **excluded from Git** per `.gitignore`. This keeps the repo small while ensuring the parsed JSON is always available.

## File Structure Details

### `data/gtfs-raw/` (Git-ignored)
- **Purpose:** Store original GTFS ZIP files
- **Size:** ~10–20 MB per file
- **Regeneration:** Download fresh feeds from:
  - UVA Transit: Transitland ID `f-university~of~virginia`
  - CAT: https://apps.charlottesville.gov/publicfiles/Charlottesvilleareatransit_gtfs.zip
  - JAUNT: Regional GTFS (contact provider)
- **When to update:** Weekly or as transit agencies release updates

### `data/gtfs-parsed/` (Git-tracked)
- **Purpose:** Parsed JSON for runtime use
- **Size:** ~1–3 MB per file (much smaller than ZIPs)
- **Format:** Stops, routes, departures by day/time
- **Generated:** `npm run parse-gtfs`
- **Used by:** `/api/score` endpoint via `lib/gtfs.ts`

## Parse Script (`scripts/parse-gtfs.js`)

The script needs to:

1. **Extract ZIP files** from `data/gtfs-raw/`
2. **Parse CSV files:**
   - `stops.txt` → Extract: `stop_id, stop_name, stop_lat, stop_lon`
   - `routes.txt` → Extract: `route_id, route_short_name, route_type, route_color`
   - `stop_times.txt` → Extract: `trip_id, stop_id, arrival_time, stop_sequence`
   - `trips.txt` → Extract: `trip_id, route_id, service_id`
   - `calendar.txt` → Extract: `service_id, monday, tuesday, ..., sunday, start_date, end_date`

3. **Build lookup tables:**
   - `departures[stop_id][day] = ["07:15", "07:45", "08:15", ...]`
   - `stop_routes[stop_id] = ["route_1", "route_2"]`

4. **Export JSON:**
   ```json
   {
     "agency": "UVA Transit Service",
     "stops": [...],
     "routes": [...],
     "departures": {...},
     "stop_routes": {...}
   }
   ```

## Runtime: How `lib/gtfs.ts` Loads Feeds

```typescript
// At server startup (once per Next.js instance)
import fs from 'fs';
import path from 'path';

const FEEDS = new Map();

function loadFeeds() {
  const feeds = ['uva-gtfs', 'cat-gtfs', 'jaunt-gtfs'];
  for (const feed of feeds) {
    const data = JSON.parse(
      fs.readFileSync(path.join(process.cwd(), `data/gtfs-parsed/${feed}.json`), 'utf-8')
    );
    FEEDS.set(feed, data);
  }
}

loadFeeds(); // Call once on server init
```

**Why this works:**
- No runtime API calls to GTFS providers
- Instant haversine calculations to find nearest stops
- Fast departure time lookups (pre-indexed by stop_id and day of week)
- Fallback gracefully if a feed is unavailable

## Troubleshooting

### "Module not found: data/gtfs-parsed/uva-gtfs.json"
**Solution:** Run `npm run parse-gtfs` to generate the JSON files from ZIPs.

### Parse script crashes on ZIP extraction
**Solution:** Ensure the ZIP files are in `data/gtfs-raw/` with exact names:
- `University_Transit_Service_GTFS.zip` (not `uva-gtfs.zip`)
- `Charlottesvilleareatransit_GTFS.zip`
- `jaunt_connect_charlottesville.zip`

### "No transit options available" in trip results
**Check:**
- Are parsed JSON files generated? (`npm run parse-gtfs`)
- Do stops exist within 400m of origin/destination in the JSON?
- Is the day of week in the calendar active (not a holiday)?

### Departures showing on Sunday when there's no Sunday service
**Root cause:** `calendar.txt` not properly parsed or service_id mismatch.
**Solution:** Verify `calendar.txt` has correct day-of-week flags. See `scripts/parse-gtfs.js` logging.
